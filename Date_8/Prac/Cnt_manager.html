üìñ Project Description

Build a Contact List Manager web application that allows users to Create, Read, Update, and Delete (CRUD) contacts. This project will test your understanding of JavaScript fundamentals, DOM manipulation, form validation, and event handling.

‚úÖ Functional Requirements

1. Contact Information

Each contact should have the following fields:

  • Name (Required)
  • Email (Required)
  • Phone Number (Required)
  • City (Optional)

2. Core Features (CRUD Operations)

CREATE - Add Contact
  • Create a form with input fields for Name, Email, Phone, and City
  • Add an "Add Contact" button
  • When clicked, validate all required fields
  • If validation passes, create a Contact object and add it to an array
  • Display the new contact in the contact list
  • Clear the form after successful addition
READ - Display Contacts
  • Display all contacts in a list format
  • Show all contact information (Name, Email, Phone, City)
  • Display a message "No contacts yet" when the list is empty
  • Show the total number of contacts
UPDATE - Edit Contact
  • Add an "Edit" button for each contact
  • When clicked, populate the form with the contact's current data
  • Change the form title to "Edit Contact"
  • Replace the "Add" button with an "Update" button
  • Add a "Cancel" button to abandon editing
  • When Update is clicked, validate and save changes
  • Visually highlight the contact being edited
DELETE - Remove Contact
  • Add a "Delete" button for each contact
  • Ask for confirmation before deleting: "Are you sure you want to delete this contact?"
  • Remove the contact from the array and update the display
  • Update the total contact count

3. Additional Features

SEARCH - Filter Contacts
  • Create a search input field
  • As the user types (keyup event), filter contacts
  • Search should work on Name, Email, or Phone fields
  • Search should be case-insensitive
  • Show all contacts when search is empty
VALIDATION - Form Validation
  • Name must not be empty
  • Email must not be empty and should contain '@' symbol
  • Phone must not be empty
  • Show error messages for invalid fields
  • Add visual indicators (red border) for fields with errors
  • Prevent form submission if validation fails
